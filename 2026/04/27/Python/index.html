一、核心语法结构

1. 单分支 if 语句

语法：if 条件表达式: （缩进的代码块，条件为True时执行）

2. 双分支 if-else 语句

语法：if 条件表达式: （条件True执行的代码块）else: （条件False执行的代码块）

3. 多分支 if-elif-else 语句

语法：if 条件1: （条件1True执行）elif 条件2: （条件1False、条件2True执行）… else: （所有条件False执行）
特点：按顺序判断，匹配第一个满足的条件后，后续条件不再执行。

二、核心规则

  • 缩进是语法一部分，同一代码块缩进需一致（推荐4个空格）。
  • 条件表达式会自动转为布尔值，False、None、0、空容器均视为False，其余为True。
  • 可通过 and（且）、or（或）、not（非）组合多个条件。
  • 支持条件语句嵌套，实现复杂逻辑。

三、特殊用法

  • 三元表达式：值1 if 条件 else 值2（条件True返回值1，否则返回值2）。
  • pass语句：用于条件块占位，避免语法错误。
  • 链式条件判断：支持连续比较运算符（如1<=x<=10），等价于多个and组合。

四、常见避坑点

  • 条件判断用==（比较），不可用=（赋值）。
  • 多分支需将具体条件放在前面，避免逻辑颠倒。
  • 注意空容器、零值的布尔判断，避免误判。

五、避免中文乱码的语法

  • 核心语法：在Python文件开头添加编码声明，指定文件编码格式。
  • 常用声明（推荐）：# coding=utf-8 或 # -- coding: utf-8 --
  • 说明：该声明需放在文件第一行（若有注释，可放在注释之后、代码之前），确保Python解释器正确识别中文字符，避免运行时出现乱码。

六、循环语法

1. for 循环

语法：for 变量 in 可迭代对象: （缩进的循环体代码）
特点：遍历可迭代对象（如列表、字符串、元组等），遍历结束后循环终止。

2. while 循环

语法：while 条件表达式: （缩进的循环体代码）
特点：只要条件表达式为True，就持续执行循环体，条件为False时终止。

3. 循环控制语句

  • break：立即终止当前循环，跳出循环体，不再执行后续循环迭代。
  • continue：跳过当前迭代的剩余代码，直接进入下一次循环迭代。
  • else：与for/while循环搭配，循环正常执行完毕（未被break终止）时，执行else后的代码块。

七、循环核心规则

  • 循环体需缩进，与条件语句缩进规则一致（推荐4个空格），缩进不一致会报错。
  • while循环需确保有终止条件，避免出现无限循环。
  • 可迭代对象为空时，for循环不执行循环体，直接结束。

八、循环特殊用法

  • range()函数：常用搭配for循环，生成指定范围的整数序列，语法：range(起始值, 终止值, 步长)（起始值默认0，步长默认1）。
  • 循环嵌套：for循环与while循环、for循环与for循环可相互嵌套，实现复杂遍历逻辑。

九、循环常见避坑点

  • while循环遗漏终止条件，导致无限循环。
  • for循环中修改可迭代对象（如列表），导致遍历异常。
  • 混淆break和continue的作用，break终止整个循环，continue仅跳过当前迭代。

十、函数语法

1. 函数定义（def）

语法：def 函数名(参数列表): （缩进的函数体代码） return 返回值（可选）
特点：有明确函数名，可复用，支持多行复杂逻辑，可添加文档说明。

2. 匿名函数（lambda）

语法：lambda 参数列表: 表达式
特点：匿名无函数名，仅一行表达式，自动返回表达式结果，适合简单临时逻辑。

3. 函数参数类型

  • 位置参数：按参数顺序传入，数量与定义时一致。
  • 默认参数：定义时指定默认值，调用时可省略该参数。
  • 可变位置参数（*args）：接收任意数量额外位置参数，打包为元组。
  • 可变关键字参数（**kwargs）：接收任意数量额外关键字参数，打包为字典。

十一、函数核心规则

  • 函数体需缩进，与条件、循环缩进规则一致（推荐4个空格）。
  • 函数参数需遵循“位置参数→默认参数→*args→**kwargs”的顺序定义。
  • 无return语句时，函数默认返回None。
  • 函数需先定义，再调用，否则会报错。

十二、函数特殊用法

  • 函数嵌套：函数内部可定义另一个函数，内部函数可访问外部函数变量。
  • 函数调用：可嵌套调用（函数内部调用其他函数），也可作为参数传递给其他函数。
  • 文档字符串：函数内部用””” “””添加说明，用于解释函数功能、参数和返回值。

十三、函数常见避坑点

  • 函数参数顺序错误，尤其是混合使用多种参数类型时。
  • 混淆lambda和def函数的适用场景，lambda仅能写单行表达式。
  • 函数内部修改外部全局变量，未用global声明导致报错。
  • 默认参数使用可变对象（如列表），导致多次调用时参数值异常。

十四、模块语法

1. 模块定义与本质

语法：将Python代码（函数、类、变量等）保存为.py文件，该文件即为一个模块。
本质：模块是Python代码的组织形式，用于实现代码复用、分类管理，避免命名冲突。

2. 模块导入方式

  • 导入整个模块：import 模块名
  • 导入模块中的指定内容：from 模块名 import 函数名/类名/变量名
  • 导入模块并指定别名：import 模块名 as 别名；from 模块名 import 内容 as 别名
  • 导入模块中所有内容（不推荐）：from 模块名 import *

十五、模块核心规则

  • 导入模块时，Python会执行模块中的所有顶层代码（函数、类定义除外，仅执行赋值、打印等语句）。
  • 模块名需符合Python命名规范（字母、数字、下划线组成，不能以数字开头）。
  • 自定义模块需与当前执行文件在同一目录，或添加模块所在路径到sys.path。
  • 内置模块（如sys、os、math）无需手动创建，直接导入即可使用。

十六、模块特殊用法

  • __name__属性：用于判断模块是被直接运行还是被导入，name == “main“时，执行模块自身的测试代码。
  • 包（package）：多个相关模块组成的目录，目录下需有__init__.py文件（可空），用于标识该目录为Python包。
  • 模块复用：同一模块可被多个程序导入，实现代码共享，减少重复编写。

十七、模块常见避坑点

  • 模块名与Python内置模块、第三方库名冲突（如命名为math、os），导致导入失败。
  • 导入自定义模块时，路径错误，导致Python无法找到模块。
  • 使用from 模块名 import * 导入时，无法导入模块中以_开头的私有内容。
  • 多次导入同一模块，Python仅执行一次模块顶层代码，不会重复执行。

十八、文件操作语法

1. 文件打开与关闭

语法：open(文件路径, 打开模式, encoding=编码格式) → 返回文件对象；关闭：文件对象.close()
简化语法（推荐）：with open(文件路径, 打开模式, encoding=编码格式) as 变量名: （缩进的文件操作代码），自动关闭文件。

2. 常用打开模式

  • r：只读模式（默认），文件不存在则报错。
  • w：写入模式，覆盖原有内容，文件不存在则创建。
  • a：追加模式，在文件末尾写入，文件不存在则创建。
  • r+：读写模式，可读取也可写入，文件不存在则报错。
  • w+：读写模式，覆盖原有内容，文件不存在则创建。
  • a+：读写模式，在文件末尾追加，文件不存在则创建。

十九、文件核心操作

  • 读取操作：read()（读取全部内容）、readline()（读取一行）、readlines()（读取所有行，返回列表）。
  • 写入操作：write(内容)（写入字符串）、writelines(列表)（写入列表中的字符串元素）。
  • 指针操作：seek(偏移量, 起始位置)，用于移动文件指针，实现指定位置读写。

二十、文件操作核心规则

  • 操作文件前必须先打开，操作完成后必须关闭（with语句可自动关闭）。
  • 写入中文时，需指定encoding参数（如encoding=”utf-8”），避免中文乱码。
  • 不同打开模式不可混用，如只读模式下无法执行写入操作。
  • 读取大文件时，推荐使用readline()或循环读取，避免占用过多内存。

二十一、文件操作特殊用法

  • 路径表示：绝对路径（完整路径）和相对路径（相对于当前执行文件的路径）均可使用。
  • 异常处理：可搭配try-except语句，捕获文件不存在、权限不足等异常，避免程序崩溃。
  • 二进制文件操作：打开模式后加”b”（如rb、wb），用于操作图片、视频等二进制文件，无需指定encoding。

二十二、文件操作常见避坑点

  • 忘记关闭文件，导致资源泄露或文件内容无法正常保存。
  • 打开模式错误，如只读模式下执行写入操作，导致报错。
  • 路径错误（拼写错误、相对路径对应错误），导致文件无法找到。
  • 写入中文未指定encoding，或编码格式不匹配，导致中文乱码。

1. 内置函数核心说明

定义：Python自带、无需导入即可直接使用的函数，覆盖数据类型转换、序列操作、逻辑判断等常用场景。
特点：无需手动定义，直接调用；语法简洁，适配日常开发高频需求。

2. 常用内置函数（按功能分类）

（1）数据类型转换类

  • int()：语法：int(参数)，将参数转为整数（支持字符串、浮点数等可转换类型）。
  • float()：语法：float(参数)，将参数转为浮点数。
  • str()：语法：str(参数)，将参数转为字符串。
  • bool()：语法：bool(参数)，将参数转为布尔值，遵循Python布尔判断规则。
  • list()：语法：list(可迭代对象)，将可迭代对象转为列表。
  • tuple()：语法：tuple(可迭代对象)，将可迭代对象转为元组。
  • dict()：语法：dict(键值对参数)，创建字典（支持关键字参数、列表/元组嵌套键值对等）。

（2）序列操作类

  • len()：语法：len(可迭代对象)，返回可迭代对象（字符串、列表、元组等）的元素个数。
  • max()：语法：max(可迭代对象)，返回可迭代对象中的最大值（需元素类型一致）。
  • min()：语法：min(可迭代对象)，返回可迭代对象中的最小值（需元素类型一致）。
  • sum()：语法：sum(可迭代对象, 起始值=0)，返回可迭代对象中所有元素的和（元素需为数值类型）。
  • sorted()：语法：sorted(可迭代对象, key=None, reverse=False)，对可迭代对象排序，返回新列表（不修改原对象）。
  • enumerate()：语法：enumerate(可迭代对象, start=0)，返回枚举对象，包含索引和对应元素。

（3）输入输出类

  • print()：语法：print(内容1, 内容2, …, sep=’ ‘, end=’\n’)，输出内容到控制台，可指定分隔符和结束符。
  • input()：语法：input(提示信息)，接收用户控制台输入，返回字符串类型。

（4）逻辑与判断类

  • abs()：语法：abs(数值)，返回数值的绝对值。
  • all()：语法：all(可迭代对象)，所有元素为True则返回True，否则返回False（空可迭代对象返回True）。
  • any()：语法：any(可迭代对象)，任意一个元素为True则返回True，否则返回False（空可迭代对象返回False）。
  • isinstance()：语法：isinstance(对象, 类型)，判断对象是否为指定类型（或类型元组），返回布尔值。

二十四、内置函数核心规则

  • 无需导入，直接调用，避免与自定义函数、变量重名，否则会覆盖内置函数功能。
  • 不同内置函数有固定参数要求，参数类型不匹配会报错。
  • 部分内置函数会返回新对象（如sorted()），不修改原可迭代对象；部分直接操作原对象（如list.append()，非内置函数，仅举例区分）。

二十五、内置函数常见避坑点

  • 自定义变量/函数名与内置函数重名（如命名为len、print），导致内置函数无法正常使用。
  • 调用max()、min()、sum()时，传入的可迭代对象元素类型不一致（如混合字符串和数值），导致报错。
  • input()接收的输入默认是字符串，直接用于数值运算时未转换类型，导致报错。
  • 混淆sorted()和列表的sort()方法，前者返回新列表，后者修改原列表。

二十六、正则表达式

1. 核心概念

  • 定义：用于匹配、查找、替换字符串的规则表达式
  • 作用：文本校验、提取内容、替换字符、格式验证

2. 常用元字符

  • .：匹配任意单个字符（换行符除外）
  • ^：匹配字符串开头
  • $：匹配字符串结尾
  • *：匹配前一个字符 0次或多次
  • +：匹配前一个字符 1次或多次
  • ?：匹配前一个字符 0次或1次
  • |：或逻辑，匹配左右任意一个表达式
  • ()：分组，提取匹配内容

3. 字符集

  • [abc]：匹配 a、b、c 中任意一个字符
  • [^abc]：匹配不是a、b、c 的字符
  • [a-z]：匹配所有小写字母
  • [0-9]：匹配所有数字

4. 预定义字符

  • \d：匹配数字，等价 [0-9]
  • \D：匹配非数字
  • \w：匹配字母、数字、下划线
  • \W：匹配非字母数字下划线
  • \s：匹配空白符（空格、换行、制表符）
  • \S：匹配非空白符

5. 数量词

  • {n}：匹配前一个字符 n次
  • {n,}：匹配至少n次
  • {n,m}：匹配 n到m次

6. Python 正则模块（re）语法

  • import re：导入正则模块
  • re.match()：从开头匹配，匹配成功返回对象
  • re.search()：全局查找第一个匹配结果
  • re.findall()：查找所有匹配结果，返回列表
  • re.sub()：替换匹配到的字符串
  • re.compile()：编译正则规则，提高效率

7. 核心规则

  • 匹配区分大小写
  • 贪婪匹配：默认匹配最长字符串
  • 非贪婪匹配：量词后加 ?，匹配最短字符串
  • 分组用 ()，可通过 group() 获取内容

8. 常见避坑点

  • 忘记转义特殊字符（. * + ? | () []）
  • 开头/结尾匹配误用 ^ 和 $
  • 贪婪匹配导致结果不符合预期
  • 中文匹配需指定 re.U 或确保编码一致

二十七、异常处理

1. 核心概念

  • 定义：捕获程序运行中的错误，避免程序崩溃，给出友好提示
  • 作用：提高程序健壮性，处理不可预知的运行异常

2. 基础语法

  • try：包裹可能出现异常的代码块
  • except：捕获指定异常，处理异常（可指定多个异常）
  • else：try代码块无异常时，执行的代码
  • finally：无论是否有异常，都必须执行的代码（释放资源）

3. 常用异常类型

  • SyntaxError：语法错误（代码编写错误）
  • TypeError：类型错误（数据类型不匹配）
  • ValueError：值错误（数据值不符合要求）
  • FileNotFoundError：文件未找到错误（文件操作时）
  • IndexError：索引错误（列表/元组索引越界）

4. 核心规则

  • except可捕获单个或多个异常（用元组包裹多个异常）
  • 可捕获所有异常（except Exception:），但不推荐（不利于排查错误）
  • 异常捕获需精准，避免滥用万能捕获

5. 常见避坑点

  • 捕获异常后未做任何处理，导致无法排查错误原因
  • 万能捕获（except:）覆盖所有异常，隐藏潜在问题
  • 混淆异常类型，导致异常无法被正确捕获

re.match只匹配字符串的开始，如果字符串开始不符合正则表达式，则匹配失败，函数返回None；而re.search匹配整个字符串，直到找到一个匹配。
